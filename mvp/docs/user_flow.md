# User Flow

## 1. 핵심 화면
- **1.0 시작 화면**: 서비스 소개 및 "질문 생성 시작" 버튼 제공  
- **2.0 주제 선정 단계**: 학습 과목/주제 선택 (드롭다운) + "다음" 버튼  
- **3.0 5W1H 요소 확인 단계**:  
  - 5개 요소(When, Where, Who, What, Why) 중 3개 이상 필수 입력 필드  
  - 미입력 시 "이 부분은 꼭 필요해요!" 안내 메시지 표시  
- **4.0 문장 완성 단계**:  
  - 자동 생성된 질문 문장 표시 + 수정 가능 텍스트 필드  
  - "학습 목적 확인" 버튼  
- **5.0 학습 목적 확인 단계**:  
  - "이 질문으로 무엇을 알고 싶나요?" 입력 필드  
  - "질문 제출" 버튼  
- **6.0 피드백 결과 화면**:  
  - 헌법 ⑦ 루브릭 기반 평가 결과 표시 (우수/양호/미흡)  
  - 재시도 또는 AI 질문 전송 유도 메시지  

## 2. 사용자 흐름 정리
```
1.0 시작 화면  
→ [질문 생성 시작] 클릭  
→ 2.0 주제 선정 단계  
→ [다음] 클릭  
→ 3.0 5W1H 요소 확인 단계  
→ 필수 요소 3개 이상 입력  
→ [다음] 클릭  
→ 4.0 문장 완성 단계  
→ 문장 수정 (선택)  
→ [학습 목적 확인] 클릭  
→ 5.0 학습 목적 확인 단계  
→ 목적 입력 후 [질문 제출] 클릭  
→ 6.0 피드백 결과 화면  
→ 결과 확인 후 재시도 또는 종료
```

## 3. 화면 전환 조건
- **조건**: 2.0에서 주제 미선택 → **전환**: "주제를 선택해 주세요" 경고 메시지 표시 후 유지  
- **조건**: 3.0에서 필수 요소 3개 미만 입력 → **전환**: "3개 이상 입력해 주세요" 안내 후 유지  
- **조건**: 5.0에서 학습 목적 미입력 → **전환**: "목적을 간단히 적어 주세요" 안내 후 유지  
- **조건**: 6.0에서 "우수" 평가 → **전환**: "AI 질문 전송" 버튼 활성화 + 학습 노트 저장 제안  
- **조건**: 6.0에서 "양호" 평가 → **전환**: "수정 후 재시도" 또는 "AI 질문 전송" 선택 제공  
- **조건**: 6.0에서 "미흡" 평가 → **전환**: "처음부터 다시 시작" 권장 메시지 표시  

## 4. 반복 사용 흐름
- **진입점**: 1.0 시작 화면 → "이전 질문 수정하기" 버튼 클릭 시 3.0 5W1H 단계로 바로 이동  
- **재사용 조건**: 이전 시도 기록이 있는 경우 3.0 단계에서 자동 입력값 복원  
- **예외 처리**: 3회 이상 "미흡" 평가 시 2.0 주제 선정 단계로 강제 복귀  

## 5. 종료 / 재시도 흐름
- **종료 조건**:  
  - 6.0 화면에서 "종료하기" 클릭 → 세션 종료 및 시작 화면으로 이동  
  - 30분 간 미활동 시 자동 종료 → "다시 질문 생성할까요?" 팝업 표시  
- **재시도 조건**:  
  - 6.0 화면에서 "다시 시도" 클릭 → 3.0 5W1H 단계로 복귀 (이전 입력값 유지)  
  - "학습 목적 확인" 실패 시 5.0 단계로만 재진입 가능
