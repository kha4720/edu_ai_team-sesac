# Build Plan

## 1. 구현 모듈
### A. 3단계 스캐폴딩 UI 모듈
- 역할: 주제 선정 → 5W1H 확인 → 문장 완성 단계별 화면 제공
- 입력값: 사용자 선택(과목/주제), 텍스트 입력(5W1H 요소, 학습 목적)
- 출력값: 단계별 입력 데이터, 다음 단계 진행 신호
- 연결 User Flow: 2.0 → 3.0 → 4.0 → 5.0

### B. 5W1H 검증 모듈
- 역할: 필수 입력 항목 실시간 검증 및 안내 메시지 표시
- 입력값: 5W1H 요소 입력값(When, Where, Who, What, Why)
- 출력값: 검증 결과(통과/미통과), 안내 메시지
- 연결 User Flow: 3.0 → 4.0 (검증 통과 시)

### C. 학습 목적 확인 모듈
- 역할: "이 질문으로 무엇을 알고 싶나요?" 확인 질문 처리
- 입력값: 사용자 텍스트 입력(학습 목적)
- 출력값: 목적 연결성 평가 결과, 다음 단계 진행 신호
- 연결 User Flow: 5.0 → 6.0

### D. 순차적 피드백 모듈
- 역할: 루브릭 기반 평가 결과 표시 및 재시도 유도
- 입력값: 5W1H 검증 결과, 학습 목적 연결성 평가 결과
- 출력값: 피드백 메시지(우수/양호/미흡), 재시도/종료 신호
- 연결 User Flow: 6.0 → (재시도 시 3.0/5.0)

## 2. 필요 API / AI 기능
### A. LLM 기반 질문 문장 생성
- 입력값: 주제, 5W1H 요소 3개 이상
- 출력값: 자동 생성된 질문 문장 (예: "과학 시험 3일 전에 생물 세포 구조를 암기하려면 어떤 방법이 효과적일까?")
- 목적: 4.0 단계에서 문장 완성 지원

### B. LLM 기반 학습 목적 연결성 평가
- 입력값: 사용자 입력 질문 문장, 학습 목적 텍스트
- 출력값: 연결성 점수(양호/미흡), 개선 제안 문구
- 목적: 5.0 단계에서 학습 목적 재확인 지원

## 3. 기술 방식
- Frontend: HTML/CSS/JavaScript (Vanilla JS) - 팀 역량 고려
- Backend: 없음 (로컬 환경에서 동작)
- AI: OpenAI GPT-4-turbo API (질문 생성/평가)
- 통신 방식: Fetch API (LLM 호출)
- 실행 환경: 브라우저 기반 (로컬 호스팅)

## 4. 구현 순서
1) 3단계 스캐폴딩 UI 모듈 구현 (프론트엔드)  
2) 5W1H 검증 모듈 구현 (프론트엔드)  
3) LLM 기반 질문 생성 API 연동 (프론트엔드)  
4) 학습 목적 확인 모듈 구현 (프론트엔드)  
5) LLM 기반 연결성 평가 API 연동 (프론트엔드)  
6) 순차적 피드백 모듈 구현 (프론트엔드)

## 5. 담당 역할
- 프론트엔드: 3명 중 2명 (UI/검증 모듈 담당)  
- 백엔드: 1명 (LLM API 연동 담당)  
- 기획/콘텐츠: 전체 팀 (5W1H 예시 문구, 피드백 메시지 작성)

## 6. 완료 기준
- 3단계 스캐폴딩 UI에서 모든 입력 필드 정상 동작  
- 5W1H 검증 모듈이 3개 이상 입력 시 다음 단계 진행 허용  
- LLM API가 주제+5W1H 입력받아 문법적으로 완전한 질문 생성  
- 학습 목적 확인 모듈에서 "양호" 이상 평가 시 피드백 결과 화면 표시  
- 순차적 피드백 모듈이 루브릭 기준에 따라 재시도/종료 유도  
- 전체 흐름이 14시간 내 시연 가능한 수준으로 구현
