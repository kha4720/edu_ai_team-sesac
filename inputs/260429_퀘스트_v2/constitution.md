# Constitution — 질문력 강화 퀘스트 서비스 교수 설계서

> **문서 성격**: 0차 산출물 (헌법). 모든 후속 산출물의 교육적 판단 기준이자 Orchestrator의 Gate 검증 기준.
> **작성 주체**: Edu Agent
> **버전**: v2.0 (Layer 2 입력용 mock — 질문력 강화 퀘스트 서비스)
> **기반**: 팀 인터뷰 합의 사항 (2026-04-28)

---

## A. 설계 근거

### 1. 교육공학적 문제 재정의

| 현재 문제 | 근본 원인 | 학습 결핍 정의 |
| --- | --- | --- |
| 학생들이 AI에 질문을 잘 하지 못한다. | 좋은 질문의 구조를 모르고, 반복 훈련 동기가 없다. | **질문 구성 능력 부족 + 지속적 훈련 동기 결여** |

표면적으로는 "AI 사용법을 모른다"로 보이지만, 본질은 두 가지다. 첫째, 자기가 무엇을 묻고 싶은지 언어로 구조화하는 능력이 부족하다. 둘째, 반복 훈련이 필요함에도 단순 반복은 지루해서 지속하지 못한다. 따라서 본 서비스는 AI 사용법 강의가 아니라, **게이미피케이션을 통해 반복 훈련을 지속하게 만드는 구조** 위에서 질문력을 키우는 경험을 제공해야 한다.

### 2. 교수 기법 탐색

| 교수 기법 | 기대 효과 | MVP 적용 난이도 |
| --- | --- | --- |
| **QFT (Question Formulation Technique)** | 질문을 구조적으로 만들고 다듬는 절차적 훈련 | 낮음 (프롬프트 중심 구현 가능) |
| **스캐폴딩 (Scaffolding)** | 쉬운 단계(객관식)에서 어려운 단계(배틀)로 점진적 발판 제공 | 낮음 (퀘스트 순서 구조로 구현) |
| **게이미피케이션 (Gamification)** | 점수·등급·콤보·배틀로 반복 동기 유지 | 낮음 (점수·상태 필드 추가로 구현) |
| **경쟁 학습 (Competitive Learning)** | AI와의 배틀로 이기고 싶다는 내재적 동기 유발 | 중간 (사전 생성 AI 질문 + 비교 평가) |
| **PBL (문제 기반 학습)** | 실제 문제 해결 프로젝트 중심 학습 | 높음 (장기 운영 필요, MVP 범위 초과) |
| **산파법 (Socratic Method)** | 되묻기 대화로 스스로 깨닫게 유도 | 높음 (대화형 인터랙션 필요, MVP 범위 초과) |

### 3. MVP 교수 기법 선정

**채택 기법: QFT + 스캐폴딩 + 게이미피케이션 + 경쟁 학습**

- **QFT**: "질문을 만드는 능력"을 직접 훈련하는 핵심 메커니즘. 상황 카드형, 질문 개선형, 배틀 퀘스트의 이론적 근거.
- **스캐폴딩**: Q1(객관식) → Q2·Q4(상황 카드) → Q3(개선형) → Q5(배틀) 순서의 근거. 판별력 → 생성력 → 비교 경쟁력으로 점진적 난이도 상승.
- **게이미피케이션**: 누적 점수, 등급, 콤보 시스템, 배틀 승리 보너스의 근거. 반복 훈련을 지속하게 만드는 동기 구조.
- **경쟁 학습**: Q5 질문 배틀의 근거. AI를 이기고 싶다는 심리가 집중도를 높이고 더 좋은 질문을 만들게 만든다.

**제외 기법: PBL, 산파법**

- **PBL**: 프로젝트 단위 장기 운영이 필요해 MVP 범위 초과.
- **산파법**: 대화형 되묻기 인터랙션이 필요해 MVP 범위 초과. 후속 확장 단계로 분리.

## B. 실행 기준 (하네스 헌법)

### 4. 서비스 전체 설계 원칙

본 서비스는 다음 다섯 가지 원칙으로 작동한다.

1. **질문을 직접 만들게 한다.** 정답을 고르는 것(Q1)은 입문 단계에만 사용하고, 본 단계는 항상 직접 작성하게 한다.
2. **쉬운 것에서 어려운 것으로 단계를 높인다.** 한 세션 안에서 Q1 → Q2 → Q3 → Q4 → Q5 순서로 난이도가 올라간다. 스테이지(등급)가 올라갈수록 AI 배틀 상대의 질문 수준도 올라간다.
3. **반복해도 새롭게 느끼게 한다.** 퀘스트 풀(유형별 10개, 총 40개)에서 매 세션마다 다른 조합을 제공한다. 이미 푼 퀘스트는 우선 제외한다.
4. **평가보다 성장과 경쟁을 강조한다.** 점수는 떨어지지 않고 누적만 된다. 미흡도 최소 점수를 받는다. 배틀에서 지더라도 다음 라운드에서 만회할 수 있다.
5. **피드백 방식은 병렬 평가다.** Q2·Q3·Q4는 구체성·맥락성·목적성 세 기준을 동시에 평가하고 종합 판정을 내린다. 순차 평가(한 번에 하나씩 보완)는 채택하지 않는다. 세션 흐름을 끊지 않기 위해서다.

### 5. 학습 목표 (Learning Objective)

> 학생이 AI에게 질문할 때, **주제** / **상황** / **원하는 답의 형태**를 포함한 질문을 스스로 작성할 수 있다. 나아가 AI가 만든 질문보다 더 좋은 질문을 만들 수 있다.

- **주체**: 학생
- **조건**: AI에게 질문하는 상황에서
- **관찰 가능한 행동 1**: 주제·상황·원하는 답의 형태 세 요소를 포함한 질문을 스스로 작성
- **관찰 가능한 행동 2**: AI가 만든 질문과 비교했을 때 루브릭 기준으로 동등하거나 더 높은 평가를 받는 질문을 작성

### 6. 평가 루브릭 (Rubric)

**평가 방식**: 병렬 평가 (구체성·맥락성·목적성 세 기준 동시 판정)

**선택 근거**: ④의 "피드백 방식은 병렬 평가" 원칙에 따른다. 세션 흐름을 끊지 않고 한 번에 종합 판정을 내리는 것이 퀘스트 서비스 구조에 적합하다.

| 평가 기준 | 우수 | 양호 | 미흡 |
| --- | --- | --- | --- |
| **구체성** (주제/대상) | 주제·대상·개념이 명확하고 범위가 좁혀져 있음<br>예) "'내 마음은 호수요'가 왜 비유 표현인지" | 주제는 있으나 범위가 모호함<br>예) "비유가 뭔지" | 주제·대상이 전혀 없음<br>예) "국어 모르겠어요" |
| **맥락성** (상황) | 상황이 구체적으로 명시됨<br>예) "국어 숙제인데" | 상황이 암시되어 있으나 불분명함<br>예) "수업 시간에 나온 건데" | 상황이 전혀 없음 |
| **목적성** (원하는 답의 형태) | 원하는 답의 형태가 명확함<br>예) "예시와 함께 설명해줘" | 원하는 것이 있으나 형태가 모호함<br>예) "도와줘" | 원하는 답의 형태가 전혀 없음 |

**종합 판정 도출 규칙**

- 3개 기준 모두 우수 → **excellent (우수)**
- 1개 이상 미흡 → **needs_work (미흡)**
- 그 외 (모두 양호 이상이지만 모두 우수는 아님) → **good (양호)**

**최소 성취 기준**: 3가지 기준 모두 양호 이상 (종합 판정 good 이상)

**설정 이유**: ④의 "질문을 직접 만들게 한다" 원칙에 따라 한 영역이라도 미흡이면 질문 구성 능력이 완성되지 않은 것으로 본다.

### 7. 루브릭 기반 서비스 플로우 원칙

#### 7-1. Q1 객관식 플로우

| 조건 | 시스템 판단 | 시스템 출력 | 사용자 다음 경험 |
| --- | --- | --- | --- |
| 정답 선택 | 정답 처리 | "정답이에요!" + 해설 + +20점 표시 | Q2로 이동 |
| 오답 선택 | 오답 처리 | "이 선택도 좋지만 더 나은 답이 있어요" + 정답 해설 + +5점 표시 | Q2로 이동 |

#### 7-2. Q2·Q3·Q4 작성형 플로우 (병렬 평가)

| 조건 | 시스템 판단 | 시스템 출력 | 사용자 다음 경험 |
| --- | --- | --- | --- |
| 종합 판정 excellent | 우수 처리 | "아주 명확해졌어요!" + 3기준 결과 + +30점 | 다음 퀘스트로 이동 |
| 종합 판정 good | 양호 처리 | "좋아졌어요!" + 3기준 결과 + +20점 | 다음 퀘스트로 이동 |
| 종합 판정 needs_work | 미흡 처리 | "한 부분이 더 명확해지면 좋겠어요" + 3기준 결과 + +10점 | 다음 퀘스트로 이동 |

> ④의 병렬 평가 원칙에 따라 재시도 없이 다음 퀘스트로 이동한다. 피드백은 한두 문장 이내로 제한한다.

#### 7-3. Q5 질문 배틀 플로우 (3라운드 2승 선취)

| 조건 | 시스템 판단 | 시스템 출력 | 사용자 다음 경험 |
| --- | --- | --- | --- |
| 사용자 질문이 AI 질문보다 루브릭 합산 점수 높음 | 라운드 승리 | "이겼어요!" + 비교 결과 + +20점 | 다음 라운드 또는 최종 결과 |
| 사용자 질문이 AI 질문과 동점 | 라운드 무승부 (AI 승리 처리) | "아쉽게 비겼어요" + 비교 결과 + +5점 | 다음 라운드 또는 최종 결과 |
| 사용자 질문이 AI 질문보다 루브릭 합산 점수 낮음 | 라운드 패배 | "이번엔 AI가 더 잘했어요" + 비교 결과 + +5점 | 다음 라운드 또는 최종 결과 |
| 2승 선취 (사용자) | 최종 승리 | "AI를 이겼어요!" + +20점 보너스 | 세션 완료 화면 |
| 3라운드 전승 (사용자) | 퍼펙트 승리 | "퍼펙트 클리어!" + 추가 +15점 | 세션 완료 화면 |
| 2승 선취 (AI) | 최종 패배 | "다음엔 이길 수 있어요!" | 세션 완료 화면 |

#### 7-4. 콤보 시스템 플로우

| 조건 | 시스템 판단 | 시스템 출력 |
| --- | --- | --- |
| Q2~Q5에서 연속 양호 이상 또는 라운드 승리 | combo_count +1 | 해당 없음 (세션 완료 시 합산) |
| 2콤보 달성 시 | 콤보 보너스 확정 | 세션 완료 화면에서 "+10 콤보 보너스" 표시 |
| 3콤보 이상 달성 시 | 콤보 보너스 확정 | 세션 완료 화면에서 "+15 콤보 보너스" 표시 |
| 미흡 판정 또는 배틀 라운드 패배 | combo_count 초기화 | 해당 없음 |

#### 7-5. 등급 임계점

| 등급 | 누적 점수 | 스테이지 특징 |
| --- | --- | --- |
| 브론즈 | 0 ~ 99점 | AI 배틀 상대: 초급 수준 질문 |
| 실버 | 100 ~ 299점 | AI 배틀 상대: 중급 수준 질문 |
| 골드 | 300 ~ 599점 | AI 배틀 상대: 고급 수준 질문 |
| 플래티넘 | 600점 이상 | AI 배틀 상대: 최고급 수준 질문 |

> 등급은 게이미피케이션 장치이며 정밀한 역량 진단 도구가 아니다. 영역별 진단, 약한 영역 분석, 맞춤 추천은 MVP 제외 범위다.

## 우선순위 원칙

1. 본 헌법은 0차 산출물로서 모든 후속 문서보다 우선한다.
2. 하위 산출물이 본 헌법과 충돌할 경우 헌법 기준을 우선 적용한다.
3. 서비스 변경 시에도 본 헌법(특히 ④~⑦)과의 정합성을 우선 검토한다.
