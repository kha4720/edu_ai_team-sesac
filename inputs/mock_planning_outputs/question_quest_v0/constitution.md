# Constitution — 질문력 강화 퀘스트 서비스 교수 설계서

> **문서 성격**: 0차 산출물 (헌법). 모든 후속 산출물(구현 명세서 4종 + pytest)의 교육적 판단 기준이자 Orchestrator의 Gate 검증 기준.
> **작성 주체**: Edu Agent (mock)
> **버전**: v0.1 (Layer 2 구축용 mock 데이터)

---

## ① 교육공학적 문제 재정의

학생들이 AI에게 질문을 잘 하지 못하는 현상을 단순한 "기능 사용법 부족"이 아니라 학습 결핍 관점에서 재정의한다.

> **막연한 질문 = 메타인지 결핍 + 언어적 구체화 능력 부족**

학생은 자신이 무엇을 알고 무엇을 모르는지 스스로 점검하지 못하고(메타인지 결핍), 그 모르는 지점을 언어로 옮겨내는 훈련도 부족하다(구체화 능력 부족). 따라서 이 서비스는 "AI 사용법 강의"가 아니라 **자기 질문을 다듬는 반복적 훈련 경험**을 제공해야 한다.

---

## ② 탐색한 교수 기법 목록

| 기법 | 핵심 아이디어 | 검토 결과 |
| --- | --- | --- |
| **QFT (Question Formulation Technique)** | 학생이 직접 질문을 만들고 다듬는 절차적 훈련 | 본 서비스 핵심에 직접 부합 |
| **스캐폴딩 (Scaffolding)** | 단계별 발판을 제공해 점진적 성장 유도 | 입문 → 본 퀘스트 단계 구조에 부합 |
| **게이미피케이션 (Gamification)** | 점수·등급·완료 보상으로 동기 유지 | 누적 점수 + 등급 시스템에 부합 |
| **산파법 (Socratic Method)** | 되묻기 대화로 스스로 깨닫게 유도 | MVP 범위 초과 (대화형 인터랙션 필요) |
| **PBL (문제 기반 학습)** | 실제 문제 해결 프로젝트 중심 학습 | MVP 범위 초과 (프로젝트 단위 운영 필요) |

---

## ③ MVP 교수 기법 선정

**채택: QFT + 스캐폴딩 + 게이미피케이션**

- **QFT**: "질문을 만드는 능력"을 직접 훈련하는 핵심 메커니즘. 질문 개선형 퀘스트의 이론적 근거가 된다.
- **스캐폴딩**: 입문 객관식 1개 → 본 개선형 2개로 이어지는 세션 구조의 근거가 된다. 학생이 갑자기 어려운 작성 과제를 마주하지 않도록 발판을 제공한다.
- **게이미피케이션**: 누적 점수와 등급 시스템의 근거가 된다. 학생이 위축되지 않고 반복적으로 참여하게 만든다.

**제외: 산파법, PBL**

- **산파법**: 대화형 되묻기·힌트 인터랙션이 필요해 MVP 범위를 초과한다. 후속 확장 단계로 분리한다.
- **PBL**: 프로젝트 단위 운영이 필요해 세션 단위(3개 퀘스트, 5~10분) MVP에 부적합하다.

---

## ④ 서비스 전체 설계 원칙

본 서비스는 다음 네 가지 철학으로 작동한다.

1. **답을 바로 주지 않고 질문을 개선하도록 유도한다.** 정답을 알려주는 것이 목표가 아니라, 학생이 자기 질문을 다듬는 경험을 쌓게 만드는 것이 목표다.
2. **평가보다 성장을 강조한다.** 점수는 떨어지지 않고 누적만 된다. 사용자가 위축되지 않도록 모든 답변에 일정 점수를 부여한다.
3. **짧은 성취를 반복적으로 경험하게 한다.** 한 세션은 5~10분 내외로 끝나며, 매 세션마다 총점과 등급이 갱신되어 즉각적인 성취감을 제공한다.
4. **모든 표현은 중학생 눈높이에 맞춘 쉬운 말로 한다.** 피드백, 안내 문구, 평가 결과 모두 어려운 어휘를 피한다.

---

## ⑤ 학습 목표 (Learning Objective)

> 중학생이 AI에게 질문할 때, **주제**·**상황**·**원하는 답의 형태**를 포함한 질문을 스스로 작성할 수 있다.

- **주체**: 중학생
- **조건**: AI에게 질문하는 상황에서
- **관찰 가능한 행동**: 주제 / 상황 / 원하는 답의 형태 세 요소를 포함한 질문을 스스로 작성

---

## ⑥ 평가 루브릭 (Rubric)

학습 목표 ⑤의 세 요소를 그대로 평가 기준으로 삼는다.

| 평가 기준 | 우수 | 양호 | 미흡 |
| --- | --- | --- | --- |
| **구체성** (무엇에 대한 질문인가) | 주제·대상·개념이 명확히 드러남 | 주제는 있으나 대상이 모호함 | 무엇에 대한 질문인지 알 수 없음 |
| **맥락성** (왜 지금 필요한가) | 숙제·시험·과제 등 상황이 명확히 드러남 | 상황 단서는 있으나 불완전함 | 상황 단서가 전혀 없음 |
| **목적성** (어떤 도움을 원하는가) | 설명·예시·풀이 등 원하는 형태가 명확함 | 원하는 답의 방향만 어렴풋이 드러남 | 무엇을 원하는지 알 수 없음 |

### 학습 목표 달성 최소 성취 기준

**3가지 기준 모두 양호 이상**

**설정 이유**: 본 서비스의 설계 원칙(④) 중 "학생이 스스로 깨닫게 하는 것을 우선"이라는 항목이 있다. 과반수 양호(2/3)로 기준을 낮추면 한 영역이 미흡한 상태로 학습 목표를 달성한 것으로 처리되어 메타인지 향상이 일어나기 어렵다. 따라서 전 영역 양호 이상을 최소 기준으로 한다.

---

## ⑦ 루브릭 기반 서비스 플로우

루브릭 ⑥의 판정 결과가 서비스 동작의 조건(if)이 된다.

### 7-1. 객관식 퀘스트 (입문)

| 조건 | 서비스 동작 | 점수 부여 |
| --- | --- | --- |
| 정답 선택 | "정답입니다!" + 왜 좋은 질문인지 짧은 해설 | +20점 |
| 오답 선택 | "이 질문도 좋지만 더 명확한 선택지가 있어요" + 정답 해설 | +5점 (참여 점수) |

> 객관식은 판별력 훈련 단계이므로 오답에도 참여 점수를 부여해 학생이 위축되지 않게 한다.

### 7-2. 질문 개선형 퀘스트 (본 단계)

| 루브릭 판정 | 서비스 동작 | 점수 부여 |
| --- | --- | --- |
| **우수** (3기준 모두 우수) | "질문이 아주 명확해졌어요!" + 어떤 점이 좋아졌는지 한 문장 피드백 | +30점 |
| **양호** (3기준 모두 양호 이상, 우수가 아님) | "좋아졌어요!" + 어떤 부분이 더 명확해졌는지 한 문장 피드백 | +20점 |
| **미흡** (1기준 이상 미흡) | "한 부분이 더 명확해지면 좋겠어요" + 어떤 영역이 부족한지 한 문장 피드백 | +10점 |

### 7-3. 세션 완료 분기

| 조건 | 서비스 동작 |
| --- | --- |
| 3개 퀘스트 모두 제출 완료 | 세션 결과 화면 표시: 이번 세션 획득 점수 / 누적 총점 / 현재 등급 |
| 누적 점수가 등급 임계점 도달 | 등급 상승 메시지: "축하해요! 이제 [등급] 단계예요" |

### 7-4. 등급 임계점

| 등급 | 누적 점수 |
| --- | --- |
| 브론즈 | 0 ~ 99점 |
| 실버 | 100 ~ 299점 |
| 골드 | 300 ~ 599점 |
| 플래티넘 | 600점 이상 |

> 등급은 게이미피케이션 장치이며, 학생의 정밀한 역량 진단 도구가 아니다. 약한 영역 식별이나 맞춤 추천은 본 MVP 범위에서 제외한다.

---

## 우선순위 원칙

1. 본 헌법은 0차 산출물로서 1차·2차 후속 문서보다 우선한다.
2. 하위 산출물이 본 헌법과 충돌할 경우 헌법 기준을 우선 적용한다.
3. 향후 기능 추가·변경 시에도 본 헌법(특히 ④ 설계 원칙, ⑤ 학습 목표, ⑥ 루브릭)과의 정합성을 우선 검토한다.
